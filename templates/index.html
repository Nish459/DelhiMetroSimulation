Delhi Metro

Below is the description of all the variable used
Max_Token in Counter + Machine combined
Limits the number of tokens available for passengers.
Prob_Card
Probability of passengers using cards instead of tokens.
metro_capacity
Maximum passenger capacity of a metro train.
metro_per_hour
Number of metro trains arriving per hour.
refresh
Time interval to refill tokens in machines.
arrival_rate
Rate at which people arrive at the metro station.
tokens_available
Current number of available tokens in the system.
hours
Duration for which the metro system is simulated.
time
Keeps track of the current time in the simulation.
num_train_in_x_hours
Number of trains arriving in a specific time period.
waitt
Calculates the number of people waiting for the next train.

About

This project simulate delhi metro system and then output the average number of people waiting for next train. The algorithm is devised so that the variables value can be changed which is taken as input from user and then observing output these input are to be optimised. For example number of tokens available must be optimised. It must not be much less as it can increase waiting queue and should not be more than required also.
